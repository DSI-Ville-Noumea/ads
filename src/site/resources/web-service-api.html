GET /api/revision/
Lists all the Revisions ordered by dateEffet desc.

idRevision long

POST /api/revision/
Saves the content as a new Revision. Revision contains the details of the revision to create and tree represents the root node of the revision to save.
Request Body
{
revision {
idRevision long
idAgent int
dateModif timestamp
dateEffet timestamp
dateDecret timestamp
description string
editing boolean
style string
canEdit boolean
}
tree {
idNoeud long
idService int
idRevision long
sigle string
label string
idTypeNoeud int
codeServi string
enfants [ NoeudDto ↺ ]
actif boolean
}
}
Response Body
[ {
idNoeud long
sigle string
message string
} ]
